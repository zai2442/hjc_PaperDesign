# 活动管理模块：任务计划

## 目标
- 基于 `docs/` 现有文档，在现有 Spring Boot + MyBatis-Plus + Spring Security + Redis 基础上，实现“活动管理模块”全链路：后台接口、发布工作流、编辑能力、审计与回滚、运营后台页面、H5 SSR 活动页、测试与压测、上线与监控方案。

## 范围边界（本次实现）
- 后端：活动 CRUD、工作流、版本与变更记录、库存原子扣减、缓存、导出、权限与审计。
- Web 运营后台：独立 React + Ant Design Pro 风格前端（ProTable/ProForm/ProDescriptions）。
- H5 SSR：独立 SSR 工程渲染活动页，包含埋点上报与首屏优化策略。
- 质量：单测覆盖、接口自动化用例、压测脚本与报告、上线 checklist/灰度/回滚/监控告警文档与示例。

## 关键决策记录（随实现更新）
- API 返回：沿用现有 `Result` 包装。
- 鉴权：沿用 `@PreAuthorize` + 服务层所有权校验（owner / SUPER_ADMIN / ADMIN / COUNSELOR）。
- 乐观锁：MyBatis-Plus `@Version` + 统一 409 冲突响应。
- 库存：Redis Lua 原子预扣减 + DB 最终一致（可扩展 MQ）。

## 风险与对策（随实现更新）
- 前端技术栈与现有 Vue 不一致：以独立子工程落地，避免影响现有 SPA。
- 压测工具差异：提供可复现实验脚本与报告模板，保证可在 CI/本地跑通。
